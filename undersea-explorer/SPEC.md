# 海底探险 - SPEC

## 概述

2~6人桌游助手，3轮制海底寻宝。玩家从潜水艇出发，投骰前进拾取宝藏，在氧气耗尽前返回潜水艇结算得分。

## 组件清单

| 组件 | 数量 | 说明 |
|------|------|------|
| 骰子 | 2个 | 6面骰，面为 1,2,3,1,2,3（点数范围2~6） |
| 氧气 | 25格 | 全局共享，每轮重置 |
| 三角形宝藏 | 8枚 | 分值0~3各2个，正面1个点 |
| 四角形宝藏 | 8枚 | 分值4~7各2个，正面2个点 |
| 五角形宝藏 | 8枚 | 分值8~11各2个，正面3个点 |
| 六角形宝藏 | 8枚 | 分值12~15各2个，正面4个点 |

## 游戏准备

1. 玩家输入昵称（2~6人）
2. 各分类宝藏分别洗混，在潜水艇下方呈S形排列：
   - 第1行：自左向右8枚三角形（正面向上，显示点数标记）
   - 第2行：自右向左8枚四角形
   - 第3行：自左向右8枚五角形
   - 第4行：自右向左8枚六角形
3. 所有玩家标记在潜水艇上
4. 氧气满格（25）

### 版图布局（缓动倾斜S形）

宝藏路径不是水平网格，而是每行采用**快-慢-快**缓动倾斜的之字形下潜路线：
- 第1行从左上向右下延伸，末端（右下角）连接第2行
- 第2行以第1行末端为右上角，向左下延伸，末端（左下角）连接第3行
- 第3行向右下延伸，第4行向左下延伸，依次类推
- 每行首尾的纵向落差大（陡峭）、中段平缓，缓动公式：`f(t) = t + 0.4·sin²(πt)·(1-2t)`
- 实现上采用绝对定位：按 `tilePos(idx)` 计算每格中心坐标，偶数行自左向右、奇数行自右向左
- 几何参数：`COLS=8, M=7, S=11, MAX_TILT=7.52, ROW_GAP=13, Y0=7, BOARD_H=84`
- `.path-board` 使用 `aspect-ratio: 100/84` 固定比例；因压实机制保证可见格数 ≤ 32，无需动态高度
- 宝藏包格（如有）紧接路径末端，使用同一套 `tilePos` 定位，视觉上无缝延续

## 游戏流程

### Phase: `setup` → `playing` → `round-end` → `game-over`

### 回合流程（当前行动玩家）

```
1. 氧气消耗阶段
   - 持有N个宝藏 → 消耗N点氧气
   - 吐司提示："玩家X 持有N个宝藏，消耗N点氧气"
   - 若氧气归零 → 触发轮结束（氧气耗尽模式）

2. 投骰阶段
   - 点击按钮投掷2个骰子
   - 显示结果（2~6）

3. 方向选择（下潜/返舱）
   - 若该玩家本轮已选择过"返舱" → 跳过选择，默认返舱
   - 否则显示【下潜】【返舱】按钮
   - 选择"返舱"后，本轮后续回合不再询问，默认返舱
   - 吐司提示选择结果

4. 移动计算
   - 有效步数 = 骰子点数 - 持有宝藏数（最小为0）
   - 若有效步数为0 → 原地不动，吐司提示
   - 下潜：向前（深处）移动有效步数格
   - 返舱：向后（潜水艇方向）移动有效步数格
   - 跳过不可落脚的格子（被其他玩家占据的格、以及上一轮被拾取后已移除的格），跳过不消耗步数
   - 步数耗尽或抵达路径末端时，停在最后一个可落脚格
   - 返舱回到潜水艇 → 直接结束行动，不触发拾取/放置

5. 行动阶段（三选一）
   - 抵达格有宝藏（未被拾取）：【拾取】或【不行动】
     - 拾取：宝藏进入背包（只显示正面点数标记，不显示分值），原位渲染圆形X标记
   - 抵达格宝藏已被拿走（有X标记）：【放置】或【不行动】
     - 放置：选择背包中1个宝藏放到该格（替换X标记）
   - 抵达格为宝藏包：【拾取】或【不行动】
     - 拾取宝藏包后，原位同样渲染圆形X标记（原则上不会出现无宝藏也无X标记的格子）
```

### 轮结束条件

**条件A：所有玩家返回潜水艇**
- 所有玩家公开背包分值，计入总分

**条件B：氧气耗尽**
- 已返回潜水艇的玩家：公开背包分值，计入总分
- 未返回的玩家：背包所有宝藏打包为"宝藏包"，放置到路径最后一个宝藏格之后
  - 宝藏包视为1个宝藏（移动惩罚和氧气消耗按1计算）
  - 宝藏包分值为内含所有宝藏分值合计
  - 被拾取后在背包中显示为特殊标记

### 轮结束结算弹窗

轮结束时弹出结算窗口，展示：
- 每位玩家本轮公开的宝藏及分值明细
- 本轮各玩家得分
- 累计总分
- 宝藏包生成信息（如有）
- 关闭弹窗后进入轮间清理，开始下一轮（或游戏结束）

### 轮间清理（含压实）

1. 本轮被拾取过的格子（`state==='taken'`，有X标记）标记为 `empty`
2. **压实**：将 `path` 和 `bundles` 中所有非空 tile 按原顺序合并，前移到 `path[0..N]`，剩余位置填 `empty`；`bundles` 数组清空
   - 效果：宝藏包并入路径、X标记格被剔除，路径始终连续无空档
   - 因压实后可见格数 ≤ 32，`.path-board` 高度固定为 `BOARD_H=84`，无需动态计算
3. 本轮新生成的宝藏包（`pendingBundles`）在结算弹窗中以文字提示，**不立即渲染**；点击"下一轮"后才在版图上显示
4. 未被拾取的宝藏保留在原位（下一轮继续使用）
5. 所有玩家回到潜水艇
6. 氧气重置为25
7. 本轮最后行动的玩家作为下一轮初始玩家
8. 重置所有玩家的"返舱"选择状态

### 游戏结束

- 累计3轮后游戏结束
- 展示积分榜单（排名 + 玩家 + R1/R2/R3 各轮得分 + 总分）
- 点击"再来一局"：清档并回到玩家设置页，**自动保留上局玩家昵称**（便于快速开局）
- 点击"返回首页"：清档并清除昵称，回到落地页

### 视觉规则

- 当前行动玩家：高亮边框（accent色）
- 宝藏正面：显示形状 + 点数标记（不显示分值）
- 被拾取格：圆形X标记
- 玩家标记：彩色圆形token，显示在对应格子上
- 氧气条：进度条形式，低于5格变红警告
- 宝藏包：特殊图标（如袋子形状）

### 交互

- 投骰按钮：仅在 `turnPhase === 'roll'` 时可点击
- 方向选择：底部弹窗显示【下潜】【返舱】
- 行动选择：底部弹窗显示可选操作
- 放置宝藏：弹窗中列出背包宝藏供选择
- 吐司提示：顶部滑入，2秒自动消失
- **移动动画**：token 以 ghost 元素逐格步进（每格 260ms），路径格使用 `tilePos` 定位
- **潜水艇出发动画**：首帧将 ghost 定位在潜水艇 SVG 底部（通过 `getBoundingClientRect` 动态计算），再逐格前进；同时移除 `.sub-tokens` 中该玩家的色点标记
- **弹窗缓存**：行动/方向弹窗内容缓存到 `_lastModalHTML`，行动条上的"等待 xxx 操作…"文字可点击重新弹出
- **玩家条滚动**：回合切换时 `scrollToCurrentPlayer` 平滑滚动使当前玩家卡片进入视口

## 吐司提示时机

1. 氧气消耗："玩家X 持有N个宝藏，消耗N点氧气（剩余M）"
2. 氧气耗尽："氧气耗尽！本轮探险结束！"
3. 投骰结果："玩家X 投出了 N"
4. 方向选择："玩家X 选择下潜/返舱"
5. 有效步数修正："携带N个宝藏，实际移动M格"
6. 原地不动："宝藏太重，无法移动！"
7. 拾取宝藏："玩家X 拾取了 [形状]宝藏"
8. 放置宝藏："玩家X 放置了 [形状]宝藏"
9. 返回潜水艇："玩家X 返回了潜水艇"
10. 轮结束结算："第N轮结束！玩家X 获得M分"
11. 宝藏包生成："玩家X 的宝藏沉入海底，形成宝藏包（价值M分）"

## 边界情况

1. 有效步数为0：原地不动，仍可执行拾取/放置（如果脚下有宝藏/X标记）
2. 下潜移动超出路径末端：停留在最后一个可落脚格（已移除的末端格会被跳过）
3. 返舱移动超出潜水艇：停在潜水艇（position = -1）
4. 前方格子全部不可落脚（被占或已移除，极端情况）：无法移动，原地不动
5. 背包为空时抵达X标记格：只能不行动
6. 宝藏包被拾取后原位渲染X标记（与普通宝藏一致，可被放置覆盖）
7. 氧气在第一个玩家行动时就耗尽：该玩家直接结算（未返回则生成宝藏包）
8. 上一轮被拾取的格本轮已移除：不渲染、移动时跳过且不消耗步数

## 技术要点

- 单文件 `index.html`，零依赖（CSS/JS 内联于 `css/style.css` 和 `js/game.js`）
- localStorage 存档（key: `undersea-explorer-state`）
- 每次状态变更后 saveState + render
- Fisher-Yates 洗牌
- 深色海洋主题配色（body 渐变：`#0f3460`(顶) → `#16213e` → `#1a1a2e`(底)，模拟深海氛围）
- 触控友好（按钮最小56px高度）
- **异步动画**：`animateToken` 返回 Promise，`executeMove` 为 async 函数，动画期间隐藏原 token 避免双影
- **屏幕状态**：全局 `screen` 变量（`'landing'` | `'setup'`）控制无存档时的界面；`setupNames` 全局数组暂存玩家昵称（跨游戏保留）
- **subCenterInBoard**：通过 `getBoundingClientRect` 动态计算潜水艇 SVG 在 `.path-board` 百分比坐标系中的位置，用于出潜/返潜动画起止点
- **渲染优化**：`render()` 重建 `#app` 前保存玩家条 `scrollLeft`，重建后恢复，避免 re-render 重置滚动位置
- **弹窗与渲染解耦**：modal 位于 `#app` 之外，弹窗操作不触发整页 render，仅通过 `refreshWaitText` 局部刷新行动条文本
