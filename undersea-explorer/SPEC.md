# 海底探险 - SPEC

## 概述

2~6人桌游助手，3轮制海底寻宝。玩家从潜水艇出发，投骰前进拾取宝藏，在氧气耗尽前返回潜水艇结算得分。

## 组件清单

| 组件 | 数量 | 说明 |
|------|------|------|
| 骰子 | 2个 | 6面骰，面为 1,2,3,1,2,3（点数范围2~6） |
| 氧气 | 25格 | 全局共享，每轮重置 |
| 三角形宝藏 | 8枚 | 分值0~3各2个，正面1个点 |
| 四角形宝藏 | 8枚 | 分值4~7各2个，正面2个点 |
| 五角形宝藏 | 8枚 | 分值8~11各2个，正面3个点 |
| 六角形宝藏 | 8枚 | 分值12~15各2个，正面4个点 |

## 游戏准备

1. 玩家输入昵称（2~6人）
2. 各分类宝藏分别洗混，在潜水艇下方呈S形排列：
   - 第1行：自左向右8枚三角形（正面向上，显示点数标记）
   - 第2行：自右向左8枚四角形
   - 第3行：自左向右8枚五角形
   - 第4行：自右向左8枚六角形
3. 所有玩家标记在潜水艇上
4. 氧气满格（25）

### 版图布局（倾斜S形）

宝藏路径不是水平网格，而是每行倾斜约5°的之字形下潜路线：
- 第1行从左上向右下延伸（5°），末端（右下角）连接第2行
- 第2行以第1行末端为右上角，向左下延伸（5°），末端（左下角）连接第3行
- 第3行向右下延伸，第4行向左下延伸，依次类推
- 实现上采用绝对定位：按 `tilePos(idx)` 计算每格中心坐标，偶数行自左向右、奇数行自右向左，逐格累加水平间距与5°垂直落差

## 游戏流程

### Phase: `setup` → `playing` → `round-end` → `game-over`

### 回合流程（当前行动玩家）

```
1. 氧气消耗阶段
   - 持有N个宝藏 → 消耗N点氧气
   - 吐司提示："玩家X 持有N个宝藏，消耗N点氧气"
   - 若氧气归零 → 触发轮结束（氧气耗尽模式）

2. 投骰阶段
   - 点击按钮投掷2个骰子
   - 显示结果（2~6）

3. 方向选择（下潜/返仓）
   - 若该玩家本轮已选择过"返仓" → 跳过选择，默认返仓
   - 否则显示【下潜】【返仓】按钮
   - 选择"返仓"后，本轮后续回合不再询问，默认返仓
   - 吐司提示选择结果

4. 移动计算
   - 有效步数 = 骰子点数 - 持有宝藏数（最小为0）
   - 若有效步数为0 → 原地不动，吐司提示
   - 下潜：向前（深处）移动有效步数格
   - 返仓：向后（潜水艇方向）移动有效步数格
   - 跳过不可落脚的格子（被其他玩家占据的格、以及上一轮被拾取后已移除的格），跳过不消耗步数
   - 步数耗尽或抵达路径末端时，停在最后一个可落脚格
   - 返仓回到潜水艇 → 直接结束行动，不触发拾取/放置

5. 行动阶段（三选一）
   - 抵达格有宝藏（未被拾取）：【拾取】或【不行动】
     - 拾取：宝藏进入背包（只显示正面点数标记，不显示分值），原位渲染圆形X标记
   - 抵达格宝藏已被拿走（有X标记）：【放置】或【不行动】
     - 放置：选择背包中1个宝藏放到该格（替换X标记）
   - 抵达格为宝藏包：【拾取】或【不行动】
     - 拾取宝藏包后，原位同样渲染圆形X标记（原则上不会出现无宝藏也无X标记的格子）
```

### 轮结束条件

**条件A：所有玩家返回潜水艇**
- 所有玩家公开背包分值，计入总分

**条件B：氧气耗尽**
- 已返回潜水艇的玩家：公开背包分值，计入总分
- 未返回的玩家：背包所有宝藏打包为"宝藏包"，放置到路径最后一个宝藏格之后
  - 宝藏包视为1个宝藏（移动惩罚和氧气消耗按1计算）
  - 宝藏包分值为内含所有宝藏分值合计
  - 被拾取后在背包中显示为特殊标记

### 轮结束结算弹窗

轮结束时弹出结算窗口，展示：
- 每位玩家本轮公开的宝藏及分值明细
- 本轮各玩家得分
- 累计总分
- 宝藏包生成信息（如有）
- 关闭弹窗后进入轮间清理，开始下一轮（或游戏结束）

### 轮间清理

1. 本轮被拾取过的格子（有X标记）在下一轮直接移除：不再渲染、不可落脚，移动时跳过
2. 未被拾取的宝藏保留在原位（下一轮继续使用）
3. 所有玩家回到潜水艇
4. 氧气重置为25
5. 本轮最后行动的玩家作为下一轮初始玩家
6. 重置所有玩家的"返仓"选择状态

### 游戏结束

- 累计3轮后游戏结束
- 展示积分榜单（排名 + 各轮得分 + 总分）

## UI布局

### 竖屏移动端优先

```
┌─────────────────────────┐
│  玩家模块（横向滚动）      │  昵称 + 背包（正面标记） + 得分
├─────────────────────────┤
│  行动模块                 │  当前玩家高亮 + 骰子区 + 操作按钮
├─────────────────────────┤
│  游戏模块                 │
│  ┌───────────────────┐  │
│  │ 🚢 潜水艇 + 氧气条  │  │
│  ├───────────────────┤  │
│  │ △ △ △ △ △ △ △ △  │  │  第1行 →
│  │ ◇ ◇ ◇ ◇ ◇ ◇ ◇ ◇  │  │  第2行 ←
│  │ ☆ ☆ ☆ ☆ ☆ ☆ ☆ ☆  │  │  第3行 →
│  │ ⬡ ⬡ ⬡ ⬡ ⬡ ⬡ ⬡ ⬡  │  │  第4行 ←
│  └───────────────────┘  │
└─────────────────────────┘
```

### 视觉规则

- 当前行动玩家：高亮边框（accent色）
- 宝藏正面：显示形状 + 点数标记（不显示分值）
- 被拾取格：圆形X标记
- 玩家标记：彩色圆形token，显示在对应格子上
- 氧气条：进度条形式，低于5格变红警告
- 宝藏包：特殊图标（如袋子形状）

### 交互

- 投骰按钮：仅在 `turnPhase === 'roll'` 时可点击
- 方向选择：底部弹窗显示【下潜】【返仓】
- 行动选择：底部弹窗显示可选操作
- 放置宝藏：弹窗中列出背包宝藏供选择
- 吐司提示：顶部滑入，2秒自动消失

## 吐司提示时机

1. 氧气消耗："玩家X 持有N个宝藏，消耗N点氧气（剩余M）"
2. 氧气耗尽："氧气耗尽！本轮探险结束！"
3. 投骰结果："玩家X 投出了 N"
4. 方向选择："玩家X 选择下潜/返仓"
5. 有效步数修正："携带N个宝藏，实际移动M格"
6. 原地不动："宝藏太重，无法移动！"
7. 拾取宝藏："玩家X 拾取了 [形状]宝藏"
8. 放置宝藏："玩家X 放置了 [形状]宝藏"
9. 返回潜水艇："玩家X 返回了潜水艇"
10. 轮结束结算："第N轮结束！玩家X 获得M分"
11. 宝藏包生成："玩家X 的宝藏沉入海底，形成宝藏包（价值M分）"

## 边界情况

1. 有效步数为0：原地不动，仍可执行拾取/放置（如果脚下有宝藏/X标记）
2. 下潜移动超出路径末端：停留在最后一个可落脚格（已移除的末端格会被跳过）
3. 返仓移动超出潜水艇：停在潜水艇（position = -1）
4. 前方格子全部不可落脚（被占或已移除，极端情况）：无法移动，原地不动
5. 背包为空时抵达X标记格：只能不行动
6. 宝藏包被拾取后原位渲染X标记（与普通宝藏一致，可被放置覆盖）
7. 氧气在第一个玩家行动时就耗尽：该玩家直接结算（未返回则生成宝藏包）
8. 上一轮被拾取的格本轮已移除：不渲染、移动时跳过且不消耗步数

### 宝藏形状绘制方案

所有宝藏板块使用内联SVG绘制，圆角风格统一：
- 三角形：圆角三角形，金色系
- 四角形（菱形）：圆角菱形，蓝色系
- 五角形：使用上述提供的圆角五角星路径，紫色系
- 六角形：圆角六边形，绿色系
- 正面显示对应数量的圆点标记（1~4个点）
- 宝藏包：袋子形状图标

## 技术要点

- 单文件 `index.html`，零依赖
- localStorage 存档（key: `undersea-explorer-state`）
- 每次状态变更后 saveState + render
- Fisher-Yates 洗牌
- 深色海洋主题配色
- 触控友好（按钮最小56px高度）
